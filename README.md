## 활용 순서 (강사용)

1. **01 → 02 → 논의메모 순서로 전체 방향 파악**
   - Full-stack 스캔 방식 이해 (HW→기구→FW→SW·AI)
   - 12일 전체 일정 확인
2. **02 파일의 각 일차별 계획 → 해당 03/04 참조**
   - Phase 1 (1~8일차): Domain별 03(HW가이드) + 04(실습워크북 Phase 1)
   - Phase 2 (9~10일차): 03(ROS1) + 04(실습워크북 Phase 2 통합실습)
   - Phase 3 (11~12일차): 05(프로젝트 가이드) 콘테스트 + 발표

## 교육 철학 요약

| 원칙 | 설명 |
|------|------|
| **실습 우선** | 이론 30% / 실습 70%, 일단 해보고 부딪히며 배움 |
| **진로 탐색** | 전문가 양성이 아닌 관련 분야 적합성 판단이 목적 |
| **실패 허용** | 트러블슈팅 과정 자체를 학습으로 인정 |
| **팀 협업** | 전 실습을 팀 단위로 수행, 역할 분담 경험 |
| **통합 경험** | 개별 센서 → 통합 시스템 → 콘테스트로 이어지는 전체 흐름 체험 |

## 주요 변경사항 (원계획 대비)

| 구분 | 원계획 (TurtleBot3) | 변경 (Jetson Xavier NX) | 교육적 효과 |
|------|---------------------|------------------------|------------|
| 메인보드 | Raspberry Pi | NVIDIA Jetson Xavier NX | GPU 탑재, AI 추론 경험 가능 |
| 구동계 | DC 모터 x 2 | 옴니휠 x 3 | 운동학 벡터 분해 학습 |
| 센서 | 제한적 | 초음파3+IR3+LiDAR+습도+화재 | 다양한 센서 융합 경험 |
| 개발 방식 | SDK 활용 | 직접 GPIO + ROS1 Melodic | 하드웨어-소프트웨어 경계 이해 |
| 설계 도구 | 없음 | EasyEDA + FreeCAD + FEA + 3D프린터 | Full-stack 스캔 경험 |
