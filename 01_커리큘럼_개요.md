# 자율주행하드웨어개발 커리큘럼 개요

## 과정 개요

| 항목 | 내용 |
|------|------|
| 과정명 | 자율주행하드웨어개발 (직업계고 채용연계형 직무교육) |
| 교육기간 | 2026.08.03(월) ~ 2026.08.14(금), 70h |
| 교육대상 | 직업계고 1~2학년 재학생 |
| 교육목적 | 미래차 모빌리티 자율주행 기술 구현 체계 파악을 통한 진로적합성 및 직무역량 조기 형성 |
| 취득역량 | 진로적합성 판단, 자율주행 하드웨어/소프트웨어 기초 이해, 통합 프로젝트 수행 경험 |

## 활용 장비 (변경 사항 반영)

원계획: TurtleBot3 기반
변경: **NVIDIA AGX Jetson 기반 자율주행 플랫폼**

| 구성 요소 | 사양 |
|-----------|------|
| 메인 컴퓨터 | NVIDIA Jetson AGX Orin (Orin NX 등) |
| 구동계 | 옴니휠 3개 + 각각 BLDC/DC 모터 + 모터 드라이버 |
| 거리 센서 | 초음파 센서 HC-SR04 x 3 |
| 근접 센서 | IR 근접센서 x 3 |
| 환경 센서 | 습도 센서, 화재 감지 센서 |
| 위치 인식 | LiDAR (RPLiDAR 또는 YDLiDAR) |
| 출력 장치 | LCD 디스플레이 |
| 입력 장치 | 무선 키보드 / 마우스 |
| 전원 | 배터리 팩 |
| S/W 플랫폼 | Ubuntu + ROS2 Humble + Python/C++ |

## 교육 철학

> **"100% 이해하지 못해도 괜찮다. 경험했다는 사실이 중요하다."**

- 직업계고 1~2학년 대상, 전문가 양성이 아닌 **진로 탐색**이 목적
- 이론은 시스템이 "왜" 동작하는지 직관적으로 이해하는 수준
- 실습은 "직접 연결하고, 읽고, 움직여보는" 경험 중심 (실습 70%, 이론 30%)
- 모든 실습은 팀 단위 협업으로 진행
- 실패 경험도 학습으로 인정, 트러블슈팅 과정 자체를 배움으로

## 학습 로드맵

```
1~2일차: 진단 및 오리엔테이션
   └─ 진로적성 진단, 현직자 특강, 기업 탐방
   
3~5일차: 자율주행 시스템 이해
   ├─ 자율주행 기술 개요 (이론+시연)
   ├─ ROS2 기초
   ├─ 센서 데이터 수집 실습 (초음파, IR, LiDAR, 습도, 화재)
   ├─ 모터 구동 실습 (옴니휠)
   └─ 센서 융합과 기본 주행

6~10일차: 미니프로젝트
   ├─ 팀 구성 및 요구사항 정의
   ├─ 자율주행 통합 기능 구현
   │  ├─ 장애물 회피
   │  ├─ 라인/경로 추종
   │  ├─ 화재 감지 및 정지
   │  └─ LCD 상태 표시
   └─ 통합 테스트 및 디버깅

11~12일차: 콘테스트 및 평가
   ├─ 주행 콘테스트
   ├─ 프로젝트 발표 자료 제작
   ├─ 최종 발표 및 강평
   └─ 수료
```

## 평가 방식

| 구분 | 비중 | 내용 |
|------|------|------|
| 진단평가 | - | 교육 전 진로적성 진단 (참고용) |
| 성취도 평가 | 30% | 각 과목별 이해도 확인 (실습 위주) |
| 프로젝트 평가 | 50% | 주행 콘테스트 결과 + 발표 평가 |
| 참여도 | 20% | 출석, 팀 기여도, 태도 |
| 취업지도 | 별도 | 교육생 80% 이상 취업 상담 이수 |
