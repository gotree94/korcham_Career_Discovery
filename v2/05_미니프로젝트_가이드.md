# 미니프로젝트 가이드

## 프로젝트 개요

자율주행 통합 기능 구현 및 주행 콘테스트

| 항목 | 내용 |
|------|------|
| 기간 | 9~12일차 (통합 실습 12h + 콘테스트 4h + 발표 4h) |
| 팀 구성 | 4~5인 x 5팀 |
| 평가 | 도메인 실습 40% + 주행 콘테스트 30% + 발표 30% |
| 전제 | 1~8일차 Full-stack Domain Scan의 결과물을 통합하여 완성 |

## Domain Scan 결과물 → 프로젝트 연결

각 분야에서 배운 내용이 어떻게 최종 프로젝트에 연결되는지:

| Domain | 결과물 | 프로젝트 활용 |
|--------|--------|-------------|
| HW (EasyEDA) | 초음파 센서 회로도/PCB 도면 | 센서 연결의 이론적 근거, "왜 이 핀에 연결했는지" 설명 자료 |
| HW (신호측정) | 오실로스코프 파형 이미지 | 센서 신호의 신뢰성 검증, 트러블슈팅 근거 |
| HW→FW (GPIO) | GPIO LED/버튼 제어 코드 | GPIO 초기화 코드, 센서 입력 처리의 기초 |
| 기구 (CAD) | 브라켓 STL 파일, 3D프린터 출력물 | 센서 장착 위치의 설계 근거, 발표 시연 자료 |
| 기구 (FEA) | 해석 전후 비교 이미지 | 설계의 타당성 증명, "왜 이 위치에 보강이 필요한지" 설명 |
| FW (PWM+모터) | 옴니휠 3개 동기 제어 코드 | 모터 드라이버 설정, PWM 제어 루틴 — 프로젝트에 직접 재사용 |
| SW·AI (Linux) | 센서 데이터 로그, 터미널 명령어 | ROS2 환경 설정, 파일 입출력, 디버깅 능력 |
| SW·AI (AI추론) | 이미지 분류 실행 경험 | 자율주행 인지 파이프라인의 개념적 이해 |

> **핵심**: 프로젝트는 새로운 내용이 아니라, 지금까지 만든 결과물을 **"하나로 연결"** 하는 과정이다.

## 주행 콘테스트 규칙

### 코스 구성

```
[시작] → [장애물구간] → [화재감지존] → [곡선주로] → [목적지]
```

- 직사각형 코스 (약 3m x 5m)
- 장애물 3~4개 배치 (박스, 콘 등)
- 화재 시뮬레이션 지점 1개 (LED 또는 불빛)
- 곡선 구간 포함

### 평가 기준 (100점)

| 항목 | 배점 | 세부 기준 |
|------|------|----------|
| 코스 완주 | 40점 | 제한시간(5분) 내 완주 (완주 실패 시 0점) |
| 장애물 회피 | 20점 | 장애물 접촉 1회당 -5점 |
| 화재 감지 | 20점 | 화재지점에서 30cm 이내 정지 + LCD 경고 표시 |
| 주행 안정성 | 20점 | 급정지/급회전 없이 부드러운 주행 |

### 감점 요소
- 장애물 접촉: -5점/회
- 코스 이탈: -10점/회
- 화재 미감지: -20점
- 시간 초과: 30초 초과 시 -5점/30초

## 필수 구현 기능

### 1. 장애물 회피
- LiDAR + 초음파 + IR 센서 융합
- 전방 40cm 이내 장애물 감지 시 회피 동작
- 회피 후 원래 경로로 복귀

### 2. 화재 감지 및 대응
- 불꽃 센서로 화재 감지
- 즉시 정지 + LCD에 "FIRE!" 표시
- 부저 경고 (선택)
- 3초 후 재주행 (또는 정지 유지 → 규칙에 따라)

### 3. LCD 상태 표시
- 실시간 주행 상태 표시
  - 1라인: 현재 동작 (DRIVING / AVOID / STOP / FIRE!)
  - 2라인: 속도 + 거리 정보

### 4. 옴니휠 정밀 제어
- 부드러운 가속/감속
- 정확한 회전각 제어
- 대각선 이동 가능

## 팀 구성 및 역할 (Domain Scan 기반)

팀원은 1~8일차 Domain Scan 경험을 바탕으로 자신이 맡을 역할을 선택한다.

| 역할 | Domain Scan 연결 | 주요 업무 |
|------|-----------------|----------|
| **HW 리드** | HW 분야 경험자 | 센서/모터/LiDAR 연결, 배선 관리, EasyEDA 회로도 참조, 전원 관리 |
| **기구 리드** | 기구 분야 경험자 | 브라켓/센서 장착 위치, FEA 해석 결과 기반 구조 보강 |
| **FW 리드** | FW 분야 경험자 → SW와 협업 | 모터 드라이버 제어, PWM 설정, GPIO 초기화 코드 통합 |
| **SW 리드** | SW·AI 분야 경험자 | ROS2 노드 구성, 센서-모터 통합, Linux 환경 관리 |
| **알고리즘** | SW·AI 분야 경험자 | 장애물 회피, 경로 추종, 화재 대응 로직 |
| **발표** | (모든 분야) | 8일간의 도메인 결과물 + 통합 결과를 스토리로 구성 |

> 인원이 부족한 팀은 HW+기구, FW+SW, SW+알고리즘 병합 가능
> 발표는 모든 팀원이 각자 맡은 분야를 발표 (1인 1파트)

## 개발 일정 (9~12일차)

### 9일차: Domain 결과물 → ROS2 통합 실습
- 전체 시스템 아키텍처 리뷰 (지금까지 배운 모든 것을 하나로)
- 센서 데이터 ROS2 토픽 발행 (2일차 신호측정 + 3일차 GPIO 코드 재사용)
- 모터 제어 ROS2 서브스크라이버 (6일차 PWM 코드 ROS2 연동)
- 통합 노드: 센서 → 판단 → 모터 한 사이클 완성

**산출물**: ROS2 통합 노드 코드, 시스템 블록도

### 10일차: 자율주행 통합 구현 + 테스트
- 장애물 회피 (LiDAR+초음파+IR 융합)
- 화재 감지 + 비상 정지
- LCD 상태 표시
- 통합 테스트 및 파라미터 튜닝
- 예행 주행

**산출물**: 통합 시스템 최종 버전, 테스트 로그, 콘테스트 준비 완료

### 11일차: 주행 콘테스트
- 코스 완주 + 장애물 회피 + 화재 감지

**산출물**: 콘테스트 결과, 피드백

### 12일차: 발표 자료 제작 + 최종 발표
- 8일간의 Domain Scan 경험 + 통합 결과를 하나의 스토리로
- 각 팀원이 자신의 역할/분야 발표

**산출물**: 발표자료, 개인별 소감문

### 메인 통합 노드 예시

```python
#!/usr/bin/env python3
import rclpy
from rclpy.node import Node
from std_msgs.msg import Float32MultiArray, String

class RobotMain(Node):
    def __init__(self):
        super().__init__('robot_main')
        
        # 상태 변수
        self.distance = 999.0
        self.ir_status = [1, 1, 1]
        self.flame_detected = False
        self.state = 'IDLE'  # IDLE, DRIVING, AVOID, FIRE_STOP
        
        # 구독
        self.create_subscription(Float32MultiArray, 'sensor_data', self.sensor_cb, 10)
        
        # 발행
        self.cmd_pub = self.create_publisher(String, 'cmd_vel_custom', 10)
        self.lcd_pub = self.create_publisher(String, 'lcd_text', 10)
        
        # 메인 루프
        self.timer = self.create_timer(0.1, self.main_loop)
    
    def sensor_cb(self, msg):
        self.distance = msg.data[0]
        self.ir_status = msg.data[1:4]
        self.flame_detected = msg.data[4] == 1.0
    
    def main_loop(self):
        if self.flame_detected:
            self.state = 'FIRE_STOP'
            self.handle_fire()
        elif self.distance < 30:
            self.state = 'AVOID'
            self.handle_avoid()
        else:
            self.state = 'DRIVING'
            self.handle_drive()
        
        # LCD 업데이트
        lcd_msg = String()
        lcd_msg.data = f"STATE:{self.state}"
        self.lcd_pub.publish(lcd_msg)
    
    def handle_drive(self):
        self.get_logger().info('전진 주행')
        # omni_drive(0, 50, 0)
    
    def handle_avoid(self):
        self.get_logger().info('장애물 회피')
        # omni_drive(0, 0, 50)  # 회전
        # time.sleep(0.5)
        # omni_drive(50, 30, 0)  # 대각선 이동
    
    def handle_fire(self):
        self.get_logger().info('🔥 화재 감지! 비상 정지!')
        # omni_drive(0, 0, 0)

def main():
    rclpy.init()
    node = RobotMain()
    rclpy.spin(node)
```

## 발표자료 가이드

### 필수 포함 내용
1. **팀 소개** (1슬라이드)
   - 팀명, 팀원, 각자 맡은 Domain 역할
2. **Full-stack Scan 경험 공유** (4슬라이드 — 팀원 1인 1슬라이드)
   - **HW 파트**: EasyEDA 회로도, 신호 측정 경험, GPIO 제어 — 내가 배운 것
   - **기구 파트**: CAD 모델링, FEA 해석 전후 비교, 3D프린터 출력물
   - **FW 파트**: PWM 모터 제어, 듀티비와 속도의 관계
   - **SW·AI 파트**: Linux 터미널, GPU 확인, AI 추론 경험
3. **통합 프로젝트** (2~3슬라이드)
   - 시스템 아키텍처 (Domain 결과물이 어떻게 연결되었는지)
   - 구현 기능: 장애물 회피, 화재 감지, 옴니휠 제어
   - 시연 영상 (1~2분, 실제 주행 모습)
4. **트러블슈팅** (1슬라이드)
   - 각 Domain별로 겪었던 문제와 해결 방법
   - "처음 해봤지만 할 수 있었다"는 경험 공유
5. **소감 및 향후 계획** (1슬라이드)
   - 개인별 소감 (팀원 각각 자신이 맡은 분야에 대해 1문장)
   - "가장 흥미로웠던 분야"와 더 알고 싶은 부분

### 발표 팁
- 10분 발표, 5분 Q&A
- 코드 설명보다는 **동작과 결과**를 중심으로
- 실패한 부분도 솔직하게 공유
- 시연 영상은 필수 (실시간 시연은 리스크 있음)
