# 논의 메모 (2026.07.03) — v2 (updated)

## 주제
직업계고 채용연계형 자율주행하드웨어개발 과정(70h) — Full-stack 스캔 방식 도입 검토

## 결정된 방향
기존 선형 커리큘럼 대신, **분야별 2~3일 집중 실습 + 통합 정리** 구조로 전환.
분야당 3일(21h) 계획을 조정하여 4개 분야 총 **8일 + 2일(통합/정리)** 로 재구성.

## 확정 하드웨어 사양
- **보드**: NVIDIA Jetson Xavier NX Developer Kit (기존 AGX Orin → 실제 확인 결과 변경)
- **CPU**: 6-core Carmel ARM v8.2 64-bit
- **GPU**: 384-core Volta + 2x NVDLA
- **RAM**: 8GB LPDDR4x
- **JetPack**: R32.6.1 (Ubuntu 18.04 aarch64)
- **ROS**: ROS1 Melodic (ROS2 Humble는 Ubuntu 22.04 필요 → JetPack 업그레이드 시 가능)

## 분야별 일정 구성

### HW 분야 (3일) — FW 일부 흡수
| 일차 | 내용 |
|------|------|
| 1일차 | EasyEDA 기본 조작 → AGX 보드 초음파 회로 따라그리기 → 풋프린트 배치 → 거버 파일 출력 |
| 2일차 | 오실로스코프/로직 애널라이저로 초음파 Trig/Echo 신호 확인, 센서 데이터시트 읽는 법 |
| 3일차 | **← FW에서 흡수**: GPIO LED/버튼 → "SW가 HW를 제어한다" 경험 |

### 기구 분야 (2일)
| 일차 | 내용 |
|------|------|
| 1일차 | FreeCAD/Onshape으로 초음파 브라켓 또는 옴니휠 마운트 모델링 |
| 2일차 | **간단 FEA 따라하기** (정하중 → 응력 집중 위치 확인 → 설계 변경 전후 비교) → 3D프린터 출력물 전달 |

\* FEA는 따라하기 수준, 수식 없이 "빨간 부분=위험→두껍게→초록" 비교로 의미 전달

### FW 분야 (1일)
| 일차 | 내용 |
|------|------|
| 1일차 | PWM+모터드라이버 핀맵 → PWM 듀티비로 속도제어 → "디지털 신호가 아날로그 동작을 만든다" |

\* GPIO 기초(LED, 버튼)는 HW 3일차로 이동, 여기서는 모터 제어에 집중

### SW·AI 분야 (2일)
| 일차 | 내용 |
|------|------|
| 1일차 | Linux 생존: 터미널 명령어, python 실행, tegrastats로 GPU 온도 확인 |
| 2일차 | 간단한 이미지 분류(TensorRT 예제) → 자율주행 차와의 연결 → 로봇산업 현황 영상/토크로 마무리 |
